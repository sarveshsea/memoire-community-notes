# Emil design engineering

Use this skill as a broad design-engineering field guide. Prefer a narrower skill such as `review-animations`, `improve-animations`, or `find-animation-opportunities` when the request has one bounded motion outcome.

## Workflow

1. Read the relevant section of [the upstream field guide](FIELD-GUIDE.md).
2. Inspect the actual interface, code, and interaction frequency before applying a rule.
3. Preserve the product's established visual language unless a change has clear user value.
4. Use exact values only when the current component and context support them.
5. Verify the result at normal speed, reduced motion, and the target input modality.

This adaptation preserves Emil Kowalski's MIT-licensed upstream guide and routes it through progressive disclosure.
