# Design systems

Read [the design-system architecture guide](references/guide.md). Inventory current product patterns and ownership before proposing a new taxonomy. Separate semantic tokens from implementation tokens, define stable component contracts, and document migration and versioning policy. Prefer measurable adoption and accessibility gates over a component-count vanity metric. Verify decisions against representative products and platforms.
